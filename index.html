04
Engineering Breakthroughs

Overcoming Real
Bugs & Constraints

Great engineering is defined by how we solve real issues. Here is how I diagnosed and debugged critical performance blockers on the GauVardhan platform.

Debugging Sync vs Async UI Caching Fallbacks Order Validation
1. Silent Order Failures & Redirect Blocks ＋
The Problem: Async Firestore calls delayed WhatsApp page redirection, triggering browser popup-blockers and leaving farmers stuck on "Placing order...".
The Solution: Restructured checkout to build the WhatsApp redirect URL synchronously at click-time, spawning the checkout window immediately while completing Firestore updates asynchronously in the background.
2. Data Loss on Intermittent Signals ＋
The Problem: Sudden drops in rural network connections wiped form data on checkout retry, leading to cart abandonment.
The Solution: Implemented local storage form caching, clear input validation rules with error prompts, and enforced business logic minimum quantities (e.g. 12 bales Pro-100) directly in the UI.
3. Code-Free Admin Price Management ＋
The Problem: Prices and stock limits fluctuated weekly. Changing these required direct code updates and redeploying the site.
The Solution: Configured a Firestore settings database matching stock/prices. Built an admin interface to push changes in real-time, with hardcoded local fallback files in case of db disconnections.
